    Namespace Aristocrat.Mgam.Client.Messaging

    Classes

    AttributesChanged

    This message will be received by the VLT when the site controller detects that an attribute registered with the current InstanceID has changed. The names and values of all changed attributes are listed. The VLT may validate the contents of the changed attributes against names and values that are expected, and if any are not valid, the VLT may send a LOCKED_SOFTWARE_ERROR notification to the site controller.

    AttributesChangedResponse

    This message is sent in response to a AttributesChanged message.

    BeginSession

    This message is sent by the VLT to the site controller to begin a session with both cash and coupon balances being zero.

    BeginSessionResponse

    This message is sent in response to a BeginSession message.

    BeginSessionWithCard

    This message is sent by the VLT to the site controller to begin a player session with a customer card.

    BeginSessionWithCash

    This message is sent by the VLT to the site controller to begin a session with cash.

    BeginSessionWithSessionId

    This message is sent by the VLT to the site controller to begin a session with a known SessionID. This should be used in off-nominal cases where the VLT has disconnected and needs to resume play with an existing session. If VoucherPrintedOffLine is true, the Site Controller will perform an implicit EndSession (VLT does not have to issue an EndSession message), and thus require the VLT to then issue a BeginSessionWithCash/Voucher if a new session is desired.

    BeginSessionWithVoucher

    This message is sent by the VLT to the site controller to begin a player session using a voucher.

    BillAcceptorMeterReport

    This message is sent by the VLT to the site controller when the cash can is pulled from the VLT. This is onlu used during Drop Mode.

    BillAcceptorMeterReportResponse

    This message is sent in response to BillAcceptorMeterReport message.

    Checksum

    This message is sent by the VLT to the site controller to send the Checksum.

    ChecksumResponse

    This message is sent in response to a Checksum message.

    ClearLock

    Clears any software locks on the VLT. Similar to a REBOOT command with the “Clear Locks” parameter, but without the rebooting.If the cause for the lock is still present (for example, paper still out) then this command may be ignored for those locks.

    ClearLockResponse

    This message is sent in response to a ClearLock message.

    ClearMeters

    VLT must SetAttribute (to 0) on all meter attributes.

    ClearMetersResponse

    This message is sent in response to a ClearMeters message.

    ComputeChecksum

    Instructs VLT to compute the checksum of the software that is currently running.

    ComputeChecksumResponse

    This message is sent in response to a ComputeChecksum message.

    CreditCash

    This message is sent by the VLT to the site controller to credit an open session with cash.

    CreditResponse

    This message is sent in response to a CreditCash message.

    CreditVoucher

    This message is sent by the VLT to credit an existing open session with a voucher.

    CustomCommand

    Custom command message.

    CustomCommandResponse

    This message is sent in response to a CustomCommand message.

    EmployeeLogin

    This message must be sent after an Employee card (see GetCardType) is inserted to get information about the employee.

    EmployeeLoginResponse

    This message is sent in response to a EmployeeLogin. It returns the name and ID of the employee.Additionally returned is a list of all Actions registered by the VLT that this employee has been granted permission to perform.

    EndSession

    This message is sent by the VLT to request the termination of an open session. If the Session balances are zero, then SRC_DoNotPrintVoucher will be returned.

    EndSessionResponse

    This message is sent in response to a EndSession message.

    EscrowCash

    This message is sent by the VLT when the bill acceptor has taken a bill into escrow, before it is stacked.

    EscrowCashResponse

    This message is sent in response to a EscrowCash message.

    Exit

    Exit the application running on the VLT. For embedded applications, the VLT should shutdown the system.All players and sessions should be logged off before issuing an exit command.This should be done by issuing a LOCK and LOGOFF_PLAYER command.

    ExitResponse

    This message is sent in response to a Exit message.

    GetAttributes

    This message will cause the site controller to send all attributes (along with their current values), to the requesting client.

    GetAttributesResponse

    This message is sent in response to a GetAttributes message.

    GetCardType

    This message is sent by the VLT to the site controller to determine what type of card is specified (i.e. Employee Card, Player Card, etc.) by any of up to three Card String Tracks.

    GetCardTypeResponse

    This message is sent in response to a GetCardType message.

    GetGameAssignments

    The VLT may optionally send this message to retrieve a list of all of the XLDF files assigned to it. This can be done in response to an Employee action, for example, for informational purposes. This message is optional, and is not needed for NYL VLT operations.

    GetGameAssignmentsResponse

    This message is sent in response to a GetGameAssignments message.

    Reboot

    Instructs the VLT to perform a reboot sequence. The VLT is required to UnregisterInstance first.

    The REBOOT command parameter may contain the phrase “Clear Lock” in which case, the VLT must clear any LOCK state.

    This message is sent in response to a Reboot message.

    Shutdown

    Perform shutdown of applications and operating system running on the VLT.All players and sessions should be logged off before issuing a shutdown command.This should be done by issuing a LOCK and LOGOFF_PLAYER command.

    If the VLT is not able to power off from software, then it must enter a state that requires manual intervention to be restarted and/or reconnected.

    ShutdownResponse

    This message is sent in response to a Shutdown message.

    Sign

    Update Message Display on VLT, for example for a progressive display.

    SignResponse

    This message is sent in response to a Sign message.

    Unplay

    Requires the VLT to return to the “Waiting for Play Command” state, as if it had just received a ReadyToPlayResponse message. Any open sessions on the VLT should be closed with a EndSession message sequence. Currently, the System will only issue this command after site closing, or in the case the system runs out of outcomes.

    UnplayResponse

    This message is sent in response to a Unplay message.
