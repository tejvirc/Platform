    Class PrintableRegion

    PrintableRegion is an area (content block) on the printed output that is self described by the syntax of the PDL (Page Description Language) and takes on the printed form of text, barcodes or graphics.

    Printer output is defined in terms of printable regions, or content blocks. Each printable region has certain required properties that describe its uniqueness.

    Remarks

    PDL Command Descriptions used for the PrintableRegion is mentioned below. The following is the structure of the DPR command. ("value" would be replaced with the appropriate property value).

    .\Hardware\Aristocrat.Monaco.Hardware\printable_regions.xml 
    
value

    • id

      Three digit printable region identifier.

      Value ranges from 000 to 999.

      000 – 099 are reserved for predefined regions.

      100 – 999 reserved for downloadable use.

    • x

      Five digit X (dot) axis start position in dots.

      Value ranges from 00000 to 65536.

    • y

      Five digit Y (paper) axis start position in dots.

      Value ranges from 00000 to 65536.

    • dx

      Five digit X (dot) axis length of the printable region in dots.

      Value ranges from 00000 to 65536.

      Note that x + dx must not overflow the width of the print head in dots; else, the content block will be rejected and a printable region truncation error will result.

    • dy

      Five digit Y (paper) axis length of printable region in dots.

      Value ranges from 00000 to 65536.

      Note that the y + dy may not exceed printable area in dots. If this is the case, the printable region will be rejected and a printable region truncation error will result.

    • rot

      One digit rotation of strings or data within printable region.

      Value ranges from 1 to 4. (1 = 0 degree, 2 = 90 degree, 3 = 180 degree and 4 = 270 degree).

    • jst

      One digit justification identifier. Defines justification of data within printable region with respect to top of printable region.

      Value ranges from 1 to 3. (1 = left justification, 2 = center justification, 3 = right justification).

    • type

      Three part printable region identifier, comprised of a letter, an equal sign, and three-digit value that points to an element in the data structure of Printer Metrics.

      The GDS Printer Communication Protocol.

      This is the print object (barcode, font or graphic) used to format the data from the command to define the template.

      1st part: F, G, or B. (F, font: Used when the printable region holds text. G, graphics: Used when printable region holds graphics. B, barcode: Used when printable region holds barcode.).

      2nd part: = (equal sign).

      3rd part: 000 to 999.

      F=005 : Points to Font [5] in data structure. G=105 : Points to Graphic [105] in data structure. B=015 : Points to Barcode Symbology [015] in data structure.

    • m1

      One digit print object multiplier 1.

      Value ranges from 1 to 6 for text or barcode and 0 for graphics.

    • m2

      Print object multiplier 2.

      Value ranges from 1 to 6 for text, 0 or 2 to 24 for barcode and 0 for graphics.

    • attr

      Three digit object printing attribute. This contains special instructions on how to treat the print objects within the printable region.

      Value ranges from 001 or 002 for text, 048 or 406 for barcode and 000 for graphics.
