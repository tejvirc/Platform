    Interface IMessageDisplay

    Definition of the IMessageDisplay interface. Together with an associated Message Display Handler object, an object that implements this interface is used to display status, error, and flower box type messages in XSpin.

    Namespace: Aristocrat.Monaco.Kernel
    Assembly: Aristocrat.Monaco.Kernel.Contracts.dll
    Syntax
    public interface IMessageDisplay
    Remarks

    In its current implementation, clients obtain an object that supports the IMessageDisplay interface from the Service Manager. The IMessageDisplay service provides the methods to register Message Display Handler objects and to notify the active handler that a message needs to be displayed or removed. Messages are wrapped in DisplayableMessage objects, which provide classification and priority information in addition to the text of the message.

    AddMessageDisplayHandler(IMessageDisplayHandler, Boolean)

    Sets the provided handler as the object to which all messages will be forwarded.

    Declaration
    void AddMessageDisplayHandler(IMessageDisplayHandler handler, bool displayPreviousMessages = true)
    Parameters
    IMessageDisplayHandler handler

    The object to set as the message handler.
    Boolean displayPreviousMessages

    Whether to display the previous soft and info messages in the MessageDisplay after the handler is added. Note: ALL hard lockups are displayed no matter what
    Remarks

    Typically, a component that implements IMessageDisplayHandler will register itself using this method. When a handler gets registered, messages previously passed to the DisplayMessage methods are forwarded to the new handler.

    Examples

    This example assumes that the class containing the SetHandler method implements the IMessageDisplayHandler interface.

    private void SetHandler()
{
IMessageDisplay messageDisplay = ServiceManager.GetInstance().GetService{IMessageDisplay}();
messageDisplay.AddMessageDisplayHandler(this);
}

    RemoveMessage(DisplayableMessage)

    Instructs the message handler to remove the message from the display.

    Declaration
    void RemoveMessage(DisplayableMessage displayableMessage)
    Parameters
    DisplayableMessage displayableMessage

    A reference to the message to remove.
    Remarks

    Use this method to remove a message displayed (or to be displayed) by the active handler. The object passed in as a parameter is used to locate the message to be removed from the service and the message handler. An exception is NOT thrown if a matching object is not found. As of this writing, object identity determines equality, so the object reference used in the RemoveMessage call must be the same as the one sent to DisplayMessage

    Examples

    This example is a continuation of the one in DisplayMessage. Assume that we already have an instance of the IMessageDisplay service and a reference to a DisplayableMessage object previously passed to DisplayMessage.

    messageDisplay.RemoveMessage(CashDoorMessage);

    RemoveMessageDisplayHandler(IMessageDisplayHandler)

    Instructs the message display to no longer forward messages to the provided handler.

    Declaration
    void RemoveMessageDisplayHandler(IMessageDisplayHandler handler)
    Parameters
    IMessageDisplayHandler handler

    The object to no longer use as a message handler.
    Remarks

    Sometimes a message display handler needs to be removed when a component that provides the message display is stopping or going out of context. For example, a game providing the message display capability goes away if the operator enters the operator menu. In this case the game message display component would need to remove itself as a message handler.

    Examples

    This example assumes that the class containing the RemoveHandler method implements the IMessageDisplayHandler interface.

    private void RemoveHandler()
{
IMessageDisplay messageDisplay = ServiceManager.GetInstance().GetService{IMessageDisplay}();
messageDisplay.RemoveMessageDisplayHandler(this);
}
