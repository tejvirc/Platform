    Interface IRtpService

    An interface by which all ReturnToPlayer (RTP) information for games can queried and validated in once centralized service.

    Namespace: Aristocrat.Monaco.Gaming.Contracts.Rtp
    Assembly: Aristocrat.Monaco.Gaming.Contracts.dll
    Syntax
    public interface IRtpService
    Remarks

    This service does not cache any RTP values or game state. Instead it acts as a set of helper methods for working with RTP information, while managing RTP calculation and RTP rules under the covers.

    The major roles of this interface are to provide a means for:

    • Querying for information about how RTP values are broken down into fine-grain contributions from various sources. For example: Base Game RTP, Progressive increment RTP, etc.
    • Querying the currently loaded RTP rules.
    • Calculating Total RTP over any set of games and WagerCategories
    • ICalculating Average RTP over any set of games and WagerCategories.
    • Automatically loading and utilizing RTP rules from jurisdictional configs to customize/restrict the RTP contribution calculations.
    • Validating that RTP information meets specific requirements set by regulators and business management.

    A breakdown of the Total RTP for the given game, into its individual contributions.
    Remarks

    Important information about RTP validation:

    • This method validates all RTP contributions represented in the breakdown and verifies progressive RTP with the progressive provider. Validation results can be reviewed by checking the IsValid and FailureFlags properties of the RtpBreakdown object that's returned.
    • This service does not disable games with invalid RTP or modify the game models in any way. Invalid games will need to be handled externally, most likely disabled or taken out of play, according to the business logic. The reason for this service not disabling games, is to help maintain a stateless service with no destructive actions.
